# CLAUDE.md - AI 에이전트 필수 규칙

> 이 파일은 모든 작업 전에 자동으로 읽힌다.
> 여기에 적힌 규칙은 선택이 아니다 **필수**다.

---

## 1단계: 당일 LOG 파일 생성

- 일자별로 logs 폴더에 yyyy-mm-dd 명칭 포맷으로 LOG 파일을 생성한다.
- 각 로그 파일은 해당 월이 지나면 logs 폴더 하위에 xx월 폴더 생성 후 파일 이동, 해당년이 지나면 xxxx년 폴더 생성 후 포함되는 폴더, 파일들 이동
- 작업 완료 시 최종 결과 요약을 1회만 로그 파일 끝에 추가한다.
- 로그 내용 포맷은 모든 내용 맨 앞에 현재 한국 시간을 "[yyyy-MM-dd HH:mm:ss] [버그수정or개선or추가] 로그 내용" 형태로 입력한다.

## 2단계: PLAN 생성

- 모든 버그수정, 개선수정, 기능추가 등 코드가 수정되는 작업에 반드시 PLAN 생성한다.
- PLAN은 간결한 내용으로 한 작업씩 목록 형태로 생성한다.
- PLAN에 없는 내용은 절대 코드수정하지 않는다.

## 3단계: 규칙

- 이미 읽은 파일은 다시 확인하지 않는다.
- 불필요한 도구 호출은 하지 않고 가능한 도구 호출은 동시에 실행한다.
- 기능마다 간략하게, 코어 로직은 상세하게 주석 추가
- 구현 혹은 수정하는 파일의 라인이 300줄 넘어가면 모듈화하여 파일당 300라인을 넘지 않도록 한다.
