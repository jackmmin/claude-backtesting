# CLAUDE.md - AI 에이전트 필수 규칙

> 이 파일은 모든 작업 전에 자동으로 읽힌다.
> 여기에 적힌 규칙은 선택이 아니다 **필수**다.

---

## 1단계: 당일 LOG 파일 생성

- 일자별로 logs 폴더에 yyyy-MM-dd 명칭 포맷으로 LOG 파일을 생성한다.
- 작업 완료 시 최종 결과 요약을 1회만 로그 파일 끝에 현재 한국 시간 "[yyyy-MM-dd HH:mm:ss] [버그수정or개선or추가] 로그 내용" 형태로 입력한다.

## 2단계: PLAN 생성

- PLAN은 간결한 내용으로 한 작업씩 목록 형태로 생성한다.
- 1~2줄 이하의 단순 수정(오타, 상수값 변경, 단일 조건 추가 등)은 PLAN 생략 가능
- 복잡한 로직 변경, 신규 기능, 버그 수정에만 PLAN 필수로 명확히 구분

## 3단계: 규칙

- 이미 읽은 파일은 다시 읽지 말고 이전 읽은 내용을 참조한다.
- 큰 파일은 offset/limit으로 필요한 부분만 읽는다.
- 불필요한 도구 호출은 하지 않고 가능한 도구 호출은 동시에 실행한다.
- 주석은 항상 꼼꼼하게 작성한다.
- 구현 혹은 수정하는 파일의 라인이 300줄 넘어가면 모듈화하여 파일당 300라인을 넘지 않도록 한다.
